# 功能清单与实现方式详解

## 一、功能总览

根据赛题要求和MiEmo论文参考，将功能分为**四个层级**：

| 层级   | 模块           | 功能数量 | 优先级     |
| ------ | -------------- | -------- | ---------- |
| **L1** | 多模态情感感知 | 5项      | P0（必须） |
| **L2** | 自适应交互引擎 | 4项      | P0（必须） |
| **L3** | 人机交互与反馈 | 6项      | P0-P1      |
| **L4** | 数据分析与报告 | 5项      | P1         |
| **L5** | 社交场景训练   | 5项      | P0         |
| **L6** | 系统配置与管理 | 4项      | P1         |

**优先级说明**：P0=Demo必须实现；P1=有最好，可降级为规划说明

## 二、模块一：多模态情感感知

### 2.1 功能清单

| ID   | 功能名称             | 描述                                   | 技术方案                               | 优先级 | 工作量 |
| ---- | -------------------- | -------------------------------------- | -------------------------------------- | ------ | ------ |
| M-01 | 实时面部表情识别     | 通过摄像头识别儿童7种基本情绪          | InsightFace / OpenFace 2.0 + MediaPipe | P0     | 中     |
| M-02 | 情绪分类输出         | 输出快乐/悲伤/愤怒/恐惧/惊讶/中性/困惑 | Softmax分类器 + 概率阈值过滤           | P0     | 低     |
| M-03 | 语音情绪分析         | 分析语音语调、节奏、音量判断情绪       | 自定义CNN-LSTM或腾讯云语音情绪API      | P0     | 中     |
| M-04 | 心率数据融合（进阶） | 通过手环获取心率、HRV评估焦虑/兴奋     | 小米/华为手环SDK                       | P1     | 高     |
| M-05 | 多模态融合决策       | 融合面部+语音+心率输出综合情绪         | 加权融合/简单投票机制                  | P0     | 中     |

### 2.2 技术实现方案（选型对比）

#### M-01：实时面部表情识别

| 方案                    | 说明                          | 优点               | 缺点              | 推荐 |
| ----------------------- | ----------------------------- | ------------------ | ----------------- | ---- |
| **OpenFace 2.0**        | 开源工具，支持AU检测+表情识别 | 学术成熟，可离线   | C++生态，部署复杂 | ⭐⭐⭐  |
| **MediaPipe Face Mesh** | Google开源，468个关键点       | 轻量、实时、多平台 | 需自己训练分类器  | ⭐⭐⭐⭐ |
| **腾讯云慧眼**          | 商业API，人脸属性分析         | 准确率高、开箱即用 | 需联网、有费用    | ⭐⭐⭐⭐ |
| **FACS自定义模型**      | 基于AU单元训练分类器          | 完全可控           | 需要标注数据      | ⭐⭐   |

**推荐方案**：MediaPipe Face Mesh + 自定义CNN分类器
- **原因**：轻量可部署到平板，数据可本地处理，避免隐私问题
- **实现路径**：MediaPipe提取面部关键点 → 输入轻量CNN（3-5层） → 输出7类情绪概率

#### M-03：语音情绪分析

| 方案                | 说明                  | 优点                    | 缺点             | 推荐 |
| ------------------- | --------------------- | ----------------------- | ---------------- | ---- |
| **OpenSMILE + SVM** | 经典语音特征提取+分类 | 学术成熟                | 特征工程复杂     | ⭐⭐⭐  |
| **CNN-LSTM端到端**  | 原始音频→频谱→分类    | 效果好，论文94.2%准确率 | 需要大量训练数据 | ⭐⭐⭐⭐ |
| **腾讯云语音情绪**  | 商业API               | 开箱即用                | 需联网、费用     | ⭐⭐⭐⭐ |

**推荐方案**：开发阶段用腾讯云API快速验证，最终用CNN-LSTM模型（可在本地运行）

#### M-05：多模态融合决策

采用**加权融合**策略：

```
综合情绪 = α × P_face + β × P_voice + γ × P_heart

预设权重（可调）：
- α = 0.5（表情权重最高）
- β = 0.3（语音辅助）
- γ = 0.2（心率参考，无硬件时自动归零）
```

## 三、模块二：自适应交互引擎

### 3.1 功能清单

| ID   | 功能名称       | 描述                              | 规则逻辑                                                  | 优先级 | 工作量 |
| ---- | -------------- | --------------------------------- | --------------------------------------------------------- | ------ | ------ |
| A-01 | 困惑检测与提示 | 检测到困惑/犹豫时自动弹出视觉提示 | 困惑概率>0.6 → 显示提示卡片                               | P0     | 低     |
| A-02 | 焦虑调节       | 检测到焦虑/高唤醒时降低难度       | 高唤醒+负效价 → 难度-1 + 播放平静音乐                     | P0     | 中     |
| A-03 | 难度动态调整   | 根据正确率和情绪调整训练难度      | 最近5次正确率>80%且情绪积极 → 难度+1；正确率<50% → 难度-1 | P0     | 中     |
| A-04 | 内容推荐       | 根据历史表现推荐下一训练内容      | 基于正确率最低的情绪类型，推荐针对性训练                  | P1     | 中     |

       ┌───────────┐      ┌───────────┐      ┌───────────┐
       │ 困惑检测   │      │ 正确/错误  │      │ 情绪检测   │
       │ >0.6?     │      │ 判断      │      │ 判断      │
       └─────┬─────┘      └─────┬─────┘      └─────┬─────┘
             │                  │                  │
             ▼                  ▼                  ▼
       ┌───────────┐      ┌───────────┐      ┌───────────┐
       │弹出提示卡片│      │正确?      │      │高唤醒+    │

### 3.3 难度分级设计

| 难度级别        | 适用场景          | 游戏参数                            |
| --------------- | ----------------- | ----------------------------------- |
| **1级（简单）** | 初次使用/焦虑状态 | 2选1选项；提示每步显示；4秒响应时间 |
| **2级（中等）** | 正常状态          | 3选1选项；逐步隐去提示；3秒响应时间 |
| **3级（困难）** | 熟练/积极状态     | 4选1选项；无自动提示；2秒响应时间   |
