# 训练内容系统升级设计

日期：2026-05-01
主题：训练内容系统升级（活动边界重构 + 多风格素材系统）
范围：情绪大侦探 / 表情连连看 / 训练素材池 / 题库结构

## 1. 目标

本次升级不是简单“多加几张图”，而是同时完成两件事情：

1. 让“情绪大侦探”和“表情连连看”真正变成两个不同的训练模块
2. 建立可持续扩展的训练素材系统，让题量和风格都能继续增长而不混乱

升级后的系统应当具备以下特征：
- 活动之间有清晰训练目标差异
- 题库不再只是同一题型的换皮复用
- 训练素材支持多风格扩展，但保持一轮训练内部风格稳定
- 后续可继续加题、加图、加第三种风格，而不需要重做整个 UI 或题库结构

## 2. 总体策略

整体产品外壳继续保持当前已确定的暖阳森林绘本 UI，不做“多风格界面系统”。变化集中发生在训练内容层：

- 界面外壳：固定统一
- 训练题型：按活动重新定义
- 训练素材：允许多风格并存
- 运行选择：系统自动控制本轮训练使用哪一种素材风格

最终原则：

> 统一产品壳，分化训练玩法，扩展多风格素材池。

## 3. 两个活动的边界重构

### 3.1 情绪大侦探

#### 核心定位
情绪大侦探的核心任务是“识别”。

孩子面对一个人物脸部、角色表情、动态刺激或复合线索时，要回答的核心问题是：

> 这个人/这个角色现在是什么情绪？

#### 训练重点
- 情绪识别
- 细节观察
- 从表情线索中做判断
- 从更复杂刺激中综合识别情绪

#### 输入 / 输出契约
- **输入**：单个刺激物（单脸、单角色、单主场景、动态片段）
- **输出**：从候选情绪中选出一个正确情绪标签
- **允许的交互**：单选判断、必要时提示后再判断
- **禁止的交互**：配对、连线、三元分组、场景与表情的双侧拖拽匹配

#### 目标对象规则
- 第一版复杂场景题不做“多人自由判断”
- 若场景复杂，仍然必须只有**一个明确主角**
- 题目 item 中必须显式写出目标对象字段，例如：
  - `target_subject_id`
  - `target_subject_role`
  - `target_subject_bbox`（如未来需要）
- 动态 clip 同样只允许一个主目标对象作为当前题目的判断对象

#### 难度分层
- **Level 1：标准表情识别**
  - 明显表情脸
  - 基础高辨识度情绪
  - 目标：基础识别

- **Level 2：弱线索识别**
  - 表情更 subtle
  - 需要看眉毛、眼睛、嘴角等细节
  - 目标：线索观察

- **Level 3：复杂/动态识别**
  - 动态刺激、连续帧、复合刺激物
  - 目标：综合判断

#### 题型模板建议
- 单脸标准表情识别
- 单脸弱表情识别
- 同一情绪不同人物/不同画风识别
- 动态短片段情绪识别
- 复合场景中的主角情绪识别

### 3.2 表情连连看

#### 核心定位
表情连连看的核心任务是“匹配”。

孩子面对表情、场景或情绪词时，要完成的是：

> 这个表情/场景/情绪词应该和谁配对？

#### 训练重点
- 建立“表情—词语—情境”之间的联系
- 把识别过的情绪放进语义和场景理解中
- 从“认出来”过渡到“连起来”

#### 输入 / 输出契约
- **输入**：两侧或多侧语义元素（表情卡、场景卡、情绪词卡）
- **输出**：完成一个或多个正确配对关系
- **允许的交互**：配对、连线、成组选择、三元关联
- **禁止的交互**：仅看单个刺激物后直接做单情绪判断作为唯一任务

#### 难度分层
- **Level 1：表情 → 情绪词**
  - 固定方向：给表情，匹配情绪词
  - 不做反向题
  - 目标：建立基础对应关系

- **Level 2：表情 → 场景 / 场景 → 表情**
  - 这是两个独立题型模板：
    - `expression_to_scene`
    - `scene_to_expression`
  - 可在同一难度层并存，但每道题只使用一种方向
  - 目标：理解表情与情境的联结

- **Level 3：场景 → 情绪词**
  - 固定方向：给场景，匹配情绪词
  - 目标：从生活情境直接抽取情绪概念

- **高阶扩展：三元匹配**
  - 表情 / 场景 / 情绪词三者成组匹配
  - 作为独立高难度模板 `triad_match`

#### triad_match 交互规则
- 第一版 `triad_match` 采用**一次只做一组**的形式
- 不做满屏多组混乱配对
- 每一题只围绕一个明确的核心线索（表情或场景）展开
- 孩子从候选项中补齐剩余两个语义元素，完成一组三元关系

#### 题型模板建议
- `expression_to_word`
- `expression_to_scene`
- `scene_to_expression`
- `scene_to_word`
- `triad_match`

## 4. 为什么它们现在会显得相似

当前版本中，这两个活动虽然页面布局略有不同，但底层题型依然非常接近：

- 都是看刺激物
- 都是从几个情绪选项里选答案
- 前端只是在 UI 上做了“普通问答”和“左图右配对”的差异化

因此升级重点不应只是换图，而必须直接改题型模板和题库结构。否则最终仍然只是：

> 同一种任务，用不同名字和不同摆法重复出现。

## 5. 素材风格系统

### 5.1 风格目标
训练素材允许存在两种风格：
- 绘本插画
- 3D 卡通

但产品界面外壳、Tino/Tina 角色系统、森林视觉世界观继续保持统一，不做整套 UI 风格混搭。

### 5.2 风格使用原则
- 多风格只发生在训练素材层
- 同一轮训练内部保持同一种风格
- 系统自动决定本轮风格
- 优先使用素材更完整的风格池
- 风格切换发生在“训练开始时”，而不是题目中途

### 5.3 风格选择策略
采用“均衡轮换”策略：

1. 开始一轮训练前，检查当前活动可用的风格池
2. 若两种风格都可用，则优先选择与上一轮不同的风格
3. 若某活动某风格素材不完整，则直接回退到完整素材池
4. 一旦本轮风格确定，中途不再切换

#### 规则细化
- “上一轮”定义为：**同一 child_id 在同一 activity_type 下完成的上一轮训练**
- 若不存在上一轮记录，则默认在可用风格里随机起始一次
- 若两种风格都可用且完整，则按“不同于上一轮”的规则优先选择
- 若只有一种风格达到完整性阈值，则忽略轮换规则，直接选完整风格
- 若两种风格都不完整，则该活动不应开放该难度层训练

该策略的目标是：
- 自动化
- 多元化
- 不让孩子在同一轮里感到风格跳变

## 6. 题库与素材的组织方式

本次升级不应只增加素材数量，而应同时引入“题型模板 + 素材池”双层结构。

### 6.1 统一数据模型
建议先锁定以下统一 schema：

#### Emotion ID
- 使用稳定英文主键：`happy / sad / angry / fear / surprise / neutral / confused`
- UI 层只负责把 ID 映射为中文词、emoji、颜色

#### Style ID
- `storybook`
- `three_d`

#### Activity ID
- `detective`
- `match`

#### Template ID
- detective:
  - `standard_face`
  - `subtle_face`
  - `cross_character`
  - `dynamic_clip`
  - `scene_focus`
- match:
  - `expression_to_word`
  - `expression_to_scene`
  - `scene_to_expression`
  - `scene_to_word`
  - `triad_match`

#### Asset ID
每个素材都应具有：
- `asset_id`
- `style_id`
- `asset_kind`（expression / scene / word / clip）
- `emotion_id`（如适用）
- `role_tags`（child, robot, adult 等）
- `difficulty_tags`
- `path`

#### Question Item Schema
第一版采用**显式完整题目**策略。每一道题都直接写清：
- `item_id`
- `activity_id`
- `template_id`
- `style_id`
- `difficulty_level`
- `stimulus_refs`
- `target_subject_id`（如适用）
- `prompt_text`（如适用）
- `correct_answer`
- `distractors`
- `match_payload`（配对题专用）
- `metadata`

不采用“运行时推导大部分字段”的方案，避免题库越写越乱。

#### Match linkage model
表情连连看素材应支持：
- 表情卡 → emotion_id
- 场景卡 → expected emotion_id
- 情绪词卡 → emotion_id
- 三元组 → `[expression_asset_id, scene_asset_id, emotion_id]`

### 6.2 情绪大侦探素材池
每个情绪大侦探题目都应从以下素材池组合生成：
- 绘本脸部素材
- 3D 卡通脸部素材
- 不同年龄/性别/外观角色
- 不同强度表情版本
- 动态素材 / 连续帧素材 / 复合刺激素材

### 6.3 表情连连看素材池
表情连连看需要维护不同语义类型的素材池：
- 表情卡片素材
- 场景插图素材
- 情绪词标签素材
- 三元对应数据组

### 6.4 数据组织建议
建议按：

`活动 × 风格 × 题型模板 × 难度`

进行组织。

例如：
- `detective / storybook / subtle_face / level-2`
- `detective / three_d / dynamic_clip / level-3`
- `match / storybook / expression_to_scene / level-2`
- `match / three_d / scene_to_word / level-3`

这样可以支持：
- 自动选择风格
- 自动按活动/难度取题
- 后续只补某一类素材，而不需要重做全部数据

## 7. 完整性判定与回退规则

### 7.1 完整性定义
“素材完整”不能是主观判断，建议使用以下规则：

完整性按**可真正抽出的可用题目数**计算，不按图片张数计算。

对某个组合：
`activity + style + template + difficulty`

当且仅当满足最低题目覆盖量时，判定为“完整”：

- detective 模板：至少 5 道预写且可抽取的可用题
- match 模板：至少 5 道预写且可抽取的可用题
- triad_match：至少 3 道预写且完整可用的三元题
- dynamic_clip：至少 3 道预写且可用的动态题

### 7.2 活动级风格可用性
某个活动在某个风格下“可用”，定义为：
- 当前活动当前难度层要求开放的所有核心模板，在该风格下都达到最低题量

### 7.3 第一版强制模板集合
为了让“可用性”可被机器判断，第一版显式规定：

#### 情绪大侦探
- Level 1：必须有 `standard_face`
- Level 2：必须有 `subtle_face`
- Level 3：必须有 `scene_focus` 或 `dynamic_clip` 至少一种

#### 表情连连看
- Level 1：必须有 `expression_to_word`
- Level 2：必须有 `expression_to_scene` 或 `scene_to_expression` 至少一种
- Level 3：必须有 `scene_to_word`
- `triad_match`：作为后续高阶扩展，第一版不作为强制开放条件

### 7.4 回退顺序
训练开始时：
1. 先看目标活动两种风格是否都完整可用
2. 若都完整，执行均衡轮换
3. 若只有一种完整，直接使用该风格
4. 若都不完整，则该活动该难度层不开放

## 8. 一轮训练中的抽题规则

### 8.1 训练开始时
系统做三件事：
1. 确认当前活动
2. 选择本轮风格（storybook / three_d）
3. 仅从该活动 + 该风格对应的题型模板和素材池中抽题

### 8.2 题型与难度推进
- 一轮训练内部风格固定
- 难度按现有会话逻辑递增或调整
- 题型切换必须遵循当前活动允许的模板集合

建议：
- **detective**：一轮中可混用多个“识别类模板”，但都必须保持“单刺激 → 单判断”的识别任务
- **match**：一轮中可混用多个“匹配类模板”，但都必须保持“多元素 → 配对/关联”的匹配任务

### 8.3 抽题顺序建议
- Level 1 优先出现基础模板
- Level 2 引入更复杂模板
- Level 3 才开放动态/三元等高阶模板

即使同一轮中模板变化，风格仍固定不变。

### 8.4 模板耗尽时的 fallback
如果一轮训练已经开始，而本轮风格下某个模板题量不足：
- 只允许切换到**本轮同风格下其它仍可用的模板**
- 不允许跨风格补题
- 不因为单个模板耗尽而直接结束整轮

### 8.5 禁止情况
- 不允许一轮中跨风格抽题
- 不允许把 detective 的识别题混入 match 的匹配轮次中
- 不允许因某模板题量不足，在轮次开始后临时切到另一风格补洞

## 9. 生图扩素材策略

### 9.1 基本原则
由于项目已有可用生图接口，素材收集不应保守。可以在题型结构已明确的前提下，**尽可能多地扩充训练素材**。

### 9.2 工作流约束
素材扩充必须遵循既定工作流：
- Claude 先根据题型模板和训练目标输出英文 prompt
- 明确风格、主体、配色、构图、背景要求、用途
- 生成后做设计审阅
- 不符合要求则继续 prompt 迭代
- 满意后才纳入正式素材池

### 9.3 扩素材优先级
推荐优先补：
1. detective Level 1 / Level 2 的基础绘本素材
2. match Level 1 / Level 2 的基础绘本素材
3. 同一批模板对应的 3D 卡通版本
4. Level 3 复杂场景与动态素材
5. triad_match 高阶扩展素材

### 9.4 扩素材目标
第一版不要求“一次补完所有题”，但应按模板稳定扩充，使每个核心模板都能逐步达到完整性阈值，并最终形成可持续增长的素材池。

## 10. 推荐实现顺序

建议按以下顺序推进：

1. 先锁定统一 schema、强制模板集合与完整性校验规则
2. 明确并重构“情绪大侦探”和“表情连连看”的题型边界
3. 改造题库结构，让题型模板成为一等公民
4. 给两个活动分别补齐绘本插画风格素材
5. 再补 3D 卡通风格素材
6. 最后接入“均衡轮换”的风格选择逻辑

这样做的原因是：
- 如果先大量生图，但题型边界和 schema 还没拆开，容易浪费素材
- 先定义玩法和数据结构，再扩素材，后面维护成本最低

## 11. 验收标准

本次升级完成后，应满足以下标准：

### 11.1 活动边界清晰
- 情绪大侦探重点在“识别”
- 表情连连看重点在“匹配”
- 两者不再只是同题型换皮

### 11.2 题型数量增加
- 每个活动至少有多类题型模板
- 难度提升体现为任务性质更复杂，而不只是选项数增加

### 11.3 素材丰富度增加
- 同一情绪不再只有极少数固定刺激图
- 至少有两种视觉风格可以支持训练扩展
- 题目扩充能持续进行，不受单次人工素材制作速度限制

### 11.4 同轮风格稳定
- 一轮训练内部素材风格统一
- 不出现一题绘本、一题 3D 的跳变感

### 11.5 系统可扩展
- 后续继续加题、加风格、加活动时，不需要推翻现有结构

## 12. 完成定义

如果升级完成后：

- 情绪大侦探和表情连连看对孩子来说是两个明显不同的训练任务
- 训练素材不再单调重复
- 每轮训练有新鲜感但仍然稳定
- 题库和素材结构可以继续扩展
- 风格选择、回退和完整性判定都可被明确实现和测试

则本次“训练内容系统升级”视为设计成功。
