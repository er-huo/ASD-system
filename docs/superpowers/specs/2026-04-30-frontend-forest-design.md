# ASD 前端视觉升级设计

日期：2026-04-30
主题：暖阳森林绘本风格 UI 重构
范围：Splash / 主页 / 训练页 / 报告页

## 1. 目标

本次改造的目标不是单纯“美化页面”，而是把当前偏功能化、偏朴素的前端界面升级为一个更适合 ASD 儿童使用与比赛展示的产品界面。

目标包括：
- 建立统一的暖阳森林绘本视觉语言
- 降低刺激强度，提升安全感、陪伴感和停留意愿
- 强化 Tino 机器人在训练页中的陪伴角色
- 提升比赛场景中的完成度、演示感和产品感
- 在视觉升级的同时保持 Flutter 交互结构可维护

## 2. 核心视觉方向

整体风格采用“暖阳森林绘本”。视觉感受应类似晨间森林中的儿童故事书，而不是通用教育后台或普通答题应用。

### 风格特征
- 主底色：奶油米白 `#FFF9F0`
- 主辅色：鼠尾草绿、暖橙棕、浅金黄、低饱和蓝绿/灰紫
- 装饰元素：树、叶子、蘑菇、草丛、小花、石头、晨光
- 卡片语言：大圆角、奶油白底、轻阴影、柔和边框
- 文案氛围：陪伴式、温柔、鼓励，而非指令式或系统式
- 动效策略：轻微淡入、上浮、呼吸感；避免闪烁、强跳跃、高对比动画

### 记忆点
产品应给人的核心印象是：

> 这不是普通儿童答题 App，而是一个住在森林晨光里的情绪训练小屋。

## 3. 页面级改造方案

### 3.1 Splash / 选人页
目标：把当前功能入口页改造成欢迎页。

设计方案：
- 顶部保留“星语灵境”标题，但采用绘本式标题区
- 中央使用宽幅森林主视觉作为核心背景
- Tino / Tina 创建入口改为奶油色角色卡，而不是普通矩形按钮
- 已有档案列表改为木牌式/故事卡片式列表
- 页面结构按“欢迎标题 → 角色选择 → 已有档案继续进入”组织

期望效果：
第一次进入时，用户感受到的是温暖和欢迎，而不是配置流程或管理界面。

### 3.2 主页 / 活动入口页
目标：从纯色色块网格升级为“森林地图中的任务小站”。

设计方案：
- 背景使用米白底 + 森林装饰元素
- 5 个活动入口改为插画卡，而不是纯色 + emoji
- 各活动保留颜色识别，但整体降饱和、弱对比
- 活动卡片配色建议：
  - 情绪大侦探：湖蓝绿
  - 表情连连看：嫩叶绿
  - 拼脸大师：灰紫
  - 社交小剧场：暖珊瑚橙
  - 心情日记：小麦黄
- 顶部加入陪伴式引导文案，例如“今天想和 Tino 一起练习什么呢？”

期望效果：
主页应像在森林里选择练习地点，而不是点功能模块。

### 3.3 训练页
目标：让 Tino 成为主陪伴角色，并降低视觉干扰。

设计方案：
- 主布局改为“左侧 Tino / 右侧任务卡”
- Tino 占屏约 20–25%，作为陪伴焦点
- 题目内容放进一张大的奶油色卡片中，统一承载题号、难度、题目、刺激物和答案区域
- 摄像头状态从高干扰角标改为低存在感状态条
- 答案按钮使用大圆角、低刺激高可读按钮样式
- Tino 继续使用 Flutter 组件绘制和动画驱动，不改成静态图片，但增加环境氛围层

期望效果：
孩子的注意力应集中在“Tino + 当前任务”上，而不是被页面碎片化信息打散。

### 3.4 报告页
目标：从系统结果页升级为温柔收尾页。

设计方案：
- 保留星级鼓励，但加入森林收尾装饰
- 情绪曲线放入更像绘本记录页的卡片中
- 主要情绪总结改写为“今天的小发现”式表达
- 回到主页按钮做得更柔和、完整、可作为结束动作
- 增加一小段陪伴式总结文案

期望效果：
报告页应让人感到完成了一段温柔旅程，而不是结束了一次系统流程。

## 4. 资产生成计划

采用“图片资产 + Flutter 组件”混合方案。

### 4.1 用图片生成的内容
这些内容适合通过 nowcoding.ai 的 `gpt-image-2` 生成：
- 森林装饰元素包
- Splash 主视觉
- 主页 5 个活动小插画
- 训练页轻背景氛围图
- 报告页收尾装饰

### 4.2 继续用 Flutter 绘制的内容
这些内容必须组件化，不应做成图片：
- 页面布局
- 按钮、输入框、卡片容器
- 星级、图表、进度条
- 题目区与答题区
- Tino 主体与情绪动画
- 摄像头状态提示

### 4.3 Tino 角色策略
Tino 在训练页保持组件化和动态化，不替换为静态图片。图片只负责：
- 背景氛围
- 角色舞台感
- 辅助装饰

最终原则：

> 图片负责氛围，Flutter 负责交互，Tino 负责陪伴。

## 5. 生图资产清单

### 第一类：通用装饰资产
- 小树
- 叶子
- 蘑菇
- 草丛
- 小花
- 石头
- 晨光装饰

### 第二类：Splash 主视觉
- 双机器人森林欢迎图
- 留出顶部标题和底部卡片布局空间
- 角色气质更接近 Tino / Tina 品牌感

### 第三类：主页活动小插画
- 情绪大侦探：放大镜 + 表情线索
- 表情连连看：成对表情卡
- 拼脸大师：五官拼图
- 社交小剧场：两个伙伴互动
- 心情日记：绘本式记录册

### 第四类：训练页背景装饰
- 左侧陪伴区森林氛围
- 右侧题目卡外缘叶片/草地装饰
- 更柔和的状态区辅助图形

### 第五类：报告页收尾装饰
- 星星
- 叶片
- 小花
- 小型完成旅程插画

## 6. 生成顺序

建议生成顺序如下：
1. Splash 主视觉
2. 主页活动插画 5 张
3. 训练页背景装饰
4. 报告页收尾装饰

该顺序可以优先提升最显眼的页面，并快速看到产品整体升级效果。

## 7. 前端结构建议

为了避免页面分别堆样式，建议新增一层轻量视觉基建：
- `ForestPalette`：统一色板
- `PageBackground`：页面通用背景层
- `DecoratedCard`：统一绘本卡片风格
- `ActivityIllustrationCard`：主页活动卡专用组件
- `SectionTitle` / `SoftStatusBar` 等局部视觉组件

这能保证四个页面共享统一设计语言，而不是每页各写一套。

## 8. 验收标准

### 8.1 风格统一
- 四个页面属于同一个产品世界观
- 配色、圆角、阴影、文案语气统一

### 8.2 ASD 友好
- 无强刺激动画
- 焦点明确
- 装饰不抢内容

### 8.3 角色陪伴感增强
- Tino 在训练页成为主陪伴者
- 情绪反馈更自然
- 进入训练时具有陪伴感

### 8.4 比赛展示效果提升
- 第一眼完成度高
- 首页与训练页更适合截图和现场演示
- 报告页更有成果感

### 8.5 实现可维护
- 图片命名清晰
- 结构可继续扩展
- 后续增加活动不需要大拆

## 9. 完成定义

如果改造完成后，产品能够让人产生如下感受，则视为本次设计成功：

> 打开 App 像进入一个温柔的森林绘本空间，孩子愿意停留，家长和评委会觉得它是经过认真打磨的康复训练产品。
